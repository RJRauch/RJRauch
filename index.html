Frontend Design

Frontend web design is an area of web development and software engineering/development(which is a field in computer science) focused on designing the interfaces of (in the case of this class) websites and web technologies; but, more generally, frontend design can involve almost any technology in which there is a human user. This involves the design of the GUI, any interactive experiences, and the visual design of the page. It focuses on the usability of the technology from the user's perspective. Frontend web design is heavily focused on the concept of usability. Broadly speaking, usabile interfaces should be effective, efficient, easy to navigate, error free, and engaging. These topics are defined below.

Effective
An effective design helps a user to achieve a desired action or accomplish a goal. Even if a design is able to accomplish the task set out by the developer, if the design does not meet the goal of the user, it can still be argued that the design is not effective.
Efficient
An efficient design minimizes the work a user has to do to achieve the desired results. It minimizes cognitive load and manual effort (keystrokes, clicks, etc.)
Easy to Navigate
Good navigational design involves a layout which is intuitively understood and implemented by the user. If the design is frustrating and confusing the user may quit interacting with the website.
Error Free
A design that is error free should involve a layout which is accessible across browsers and devices and screen readers so that it will be displayed without errors A good error free design should also anticipate how users would interact with it, and generate common errors, and handle those appropriately.
Engaging
An engaging layout and experience is one that captivates the user and consumes their attention. It is pleasant and fun and easy to use. It anticipates user needs and minimizes any roadblocks they may experience, or handles them appropriately. From a psychological perspective this can be best be summarized by a design that places a user in the psychological state of flow.

Page layout tags help determine how content is arranged and spaced on a page to help machines and screen readers understand where content is distributed on a page. Perviously developers often used "div" tags to arrange content, but this often did not easily allow for content to be differentiated by its purpose and layout.

Anchor tags really bring the concept of the web to life. What anchor tags do is allow for different pages (or websites) to be linked to each other, so that when a user clicks on the link in the browser it will take the user to the linked webpage specified in the anchor tag. Anchor tags link to content by using the "href=" attribute, which specifies the URL of the desired content. An anchor tag can also link to a position within a document by referencing a hastag with the name of the ID of an element in the document at the end of a URL. This will effectively take the user to that portion of the document specified by the ID.
